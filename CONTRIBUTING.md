# Contributing

Thanks for your interest in Kaels Map Tool.

## Use and share

You are welcome to use and share **unmodified** copies of the app. See [LICENSE](LICENSE) for details.

## Want something changed?

**Prefer a feature request.** Open an [issue](https://github.com/KaelKodes/Kaels-Map-Tool/issues) describing what you want and why. I review requests and add approved features to the official project.

## Want to write code?

Contributions are welcome through the **official repository only**:

1. Open an issue first for anything non-trivial (bug reports and tiny fixes can skip this).
2. Create a **branch** on this repo (fork → branch → pull request is fine).
3. Open a **pull request** against `main` with a clear description of the change.
4. Do **not** publish modified builds or long-lived public forks as alternate products. The license requires changes to flow back here unless you have written permission.

I aim to keep one official version so everyone playing together stays in sync.

## Bug reports

Include:

- What you expected
- What happened
- Steps to reproduce
- Godot or build version (e.g. A0.0.1)

## Questions

Use GitHub Issues with the question label, or open a discussion if discussions are enabled on the repo.
